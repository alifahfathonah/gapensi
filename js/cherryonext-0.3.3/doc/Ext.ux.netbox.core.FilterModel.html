Class Ext.ux.netbox.core.FilterModel

Defined in FilterModel.js

The filter model is the core class of the whole filter design, and probably the one you will interact most. It's the class that mantains the filter, it's the class used to obtain an export of the filter in a predefined way (look at getFilterObj for more information about the format), to import a filter, to add or remove a filter and so on. All the views (for example QuickFilterModelView, StaticFilterModelView and DynamicFilterModelView) just shows the data contained in the filter model, and use the filterModel to perform some operation on the filter. A filter model only manages elementary filters that have their field inside the fieldManager that is used to build the filterModel.

Examples

Build a filter model given the fieldManager 
var fieldManager=new Ext.ux.netbox.core.FieldManager(fieldManager);
Build a filter model using the config to a fieldManager: 
 var filterCfg=[
   {id: 'company',label: 'Company'},
   {id: 'price',label: 'Price', type: 'float'},
   {id: 'change',label: 'Change', type: 'float'},
   {id: 'pctChange',label: '% Change', type: 'float'},
   {id: 'lastChange',label: 'Last Updated', type: 'date', format: 'd/m/Y H:i'},
   {id: 'shouldBuy',label: 'Should Buy', type: 'enum', availableValues: availableValuesStore, remoteStore: false}
 ];
 var fieldManager=new Ext.ux.netbox.core.FieldManager(filterCfg);

getFilterObj

Object getFilterObj(evenInvalid, additionalFilterObj, additionalLogicalOper)
    This method returns an object representing the actual filter or null if is not setted. 
    Filter::=ElementaryFilter|CompositeFilter
ElementaryFilter::={fieldId : "<fieldId>", operatorId : "<operatorId>", values : [...]}
CompositeFilter::={left : Ext.ux.netbox.core.Filter, logicalOperator : String, right : Ext.ux.netbox.core.Filter}
logicalOperator::="AND"|"OR"

    Example 

    {
   left:{
     left:{fieldId:"field4",operatorId:"DATE_EQUAL",values:[{label:"12/10/2009 12:12",value:"2009-10-12 12:12:00"}]},
     logicalOperator:"AND",
     right:{fieldId:"field3",operatorId:"STRING_EQUAL",values:[{label:"label1",value:"valore1"}]}
   },
   logicalOperator:"AND",
   right:{fieldId:"field",operatorId:"NUMBER_EQUAL",values:[{label:5,value:5}]}
}
    Parameters:
      boolean evenInvalid - (optional) default false
      Object additionalFilterObj - (optional)
      String additionalLogicalOper - Ext.ux.netbox.core.CompositeFilter.AND/OR (optional)
    Returns:
      Object the filter object
