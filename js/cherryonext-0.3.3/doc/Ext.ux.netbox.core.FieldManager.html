Class Ext.ux.netbox.core.FieldManager

Defined in FieldManager.js

This class manages the filter fields. You should use an instance of this class (obtained using filterModel.getFieldManager) to add or remove filter fields, or to obtain them to add or remove operators. More than one filter model can have the same FieldManager, allowing to have different sets of filters, but on the same set of fields.

Ext.ux.netbox.core.FieldManager

Ext.ux.netbox.core.FieldManager(config)
    Creates a new FieldManager class in the Netbox Filter Framework, eventually instantiating the fields.
    NB: If availableValues is specified and the type of the field is string the STRING_LIST and STRING_NOT_IN_LIST operators will be added to the fields.
    Parameters:
      Array of field config config - An array of configuration object for the fields in this object. Optional, if not present an empty FieldManager is created.
      String config.id - The id of the field (for example the id of the column)
      String config.label - The label of the field (for example the header of the column)
      Array config.defaultValues - The default values of the field in the format {value: ... , label: ...} Optional
      String config.type - The type of the field. Available values are: string, enum, float, int, date. Default type is string
      String config.format - Only for dates, the format of the date. Look Ext.ux.netbox.data.DateFilterType for more details
      Ext.data.Store config.availableValues - The store containing the available values for this field. Look at Ext.ux.netbox.core.Field.setAvailableValues for more details
      boolean config.remoteStore - True if the store is remote, false otherwise. Look at Ext.ux.netbox.core.Field.setStoreRemote for more details
      boolean config.forceReload - True if you want reload the store everytime expand the combo. Look at Ext.ux.netbox.core.Field.setForceReload for more details
      boolean config.caseSensitive - True if the value should be compared with the store's one with case sensitive. Look at Ext.ux.netbox.core.Field.setCaseSensitive for more details
      function config.validate - The function used to validate the values of the Field. Look at Ext.ux.netbox.core.Field.setValidateFn for more details
